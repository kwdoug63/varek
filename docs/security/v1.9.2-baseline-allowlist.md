# v1.9.2 Baseline Allowlist — Rationale

The Warden baseline moves from **allow-by-default + denylist** (where every
un-enumerated bypass is a hole) to **default-deny allowlist** (where the residual
is denied by construction). This note records why each tier is shaped the way it
is, so the allowlist is auditable rather than folklore.

## Tiers

- **ADMIT (`SCMP_ACT_ALLOW`)** — syscalls a mediated workload issues that open no
  capability: memory management, I/O on already-authorized fds, poll/wait, time,
  signals, futex. These are the high-frequency *use* path; they are not mediated
  per call because the capability they ride was authorized at acquisition.
- **MEDIATE (`SCMP_ACT_NOTIFY`)** — capability-*acquisition* syscalls whose
  decision needs the pointer/path argument: `openat`, `openat2`, `execve`,
  `execveat`, `connect`, `bind`, `socket`, and the message syscalls. The
  supervisor decides on a copied, validated value and performs-and-ADDFDs
  (v1.9.1 discipline). `openat2` is preferred so the supervisor can demand
  `RESOLVE_NO_SYMLINKS | RESOLVE_BENEATH | RESOLVE_NO_MAGICLINKS`.
- **HARD-DENY (`SCMP_ACT_KILL_PROCESS` in strict mode)** — never admissible.
  Reaching one is an attack, not a probe.
- **DEFAULT (`SCMP_ACT_ERRNO(EPERM)`)** — everything not named. Benign probing of
  an un-admitted syscall degrades; it is not fatal.

## Why scalar-arg filtering is done here and pointer-arg is not

seccomp snapshots **register** arguments, so a decision on a scalar (e.g. the
`clone`/`unshare` flags word) is race-free and lives in this file. A decision on
**pointer-referenced memory** (a path, a struct) is *not* safe here — a second
target thread can rewrite it between check and use. Those go to the supervisor
(MEDIATE), or, where the pointer is a flags struct we cannot inspect at all
(`clone3`), the syscall is hard-denied.

## Class-to-syscall map

| Class | Syscalls handled here | Action |
|-------|----------------------|--------|
| 1 ABI | (all non-native) | DEFAULT deny via native-only filter; secondary arches removed |
| 2 variants | `clone3`, un-admitted `*2` variants | hard-deny / DEFAULT deny |
| 3 off-path | `io_uring_*`, `process_vm_readv/writev`, `pidfd_getfd`, `memfd_create` | hard-deny |
| 4 TOCTOU | `userfaultfd`, `mount`, `umount2`, `fsopen`, `fsconfig`, `fsmount`, `move_mount`, `open_tree` | hard-deny |
| 5 priv-surface | `ptrace`, `setns`, `clone3`; `clone`/`unshare` with `CLONE_NEWUSER`/ns bits | hard-deny / scalar-flag deny |
| 6 kernel/dev | `bpf`, `init_module`, `finit_module`, `delete_module`, `kexec_load`, `kexec_file_load`, `perf_event_open`, `keyctl`, `add_key`, `request_key`, `modify_ldt` | hard-deny |
| device files | (the `open` of `/dev/mem`, `/proc/kcore`, raw block devs) | contained by MEDIATE on `openat`/`openat2` |

## Tightening guidance

The ADMIT set is a **minimal baseline**, deliberately superset-free of anything
capability-opening. For a given workload, remove what it never calls; never *add*
a hard-deny syscall to ADMIT to make an error go away — if a workload appears to
need `ptrace`/`bpf`/a namespace, that is a design review, not an allowlist edit.
Narrow `ioctl` to specific request codes (it is intentionally omitted from
ADMIT). Admit `memfd_create` only with an explicit reason recorded.

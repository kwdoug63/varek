# VAREK v1.10 Architecture Roadmap

The v1.9.2 patch closes the bypass classes that a baseline filter can close by
denial and structural containment. The items below *change the enforcement model
or the trusted base*, so they are a v1.10 track rather than a drop-in patch. Each
is stated as a target with the soundness argument it must preserve. They are
sequenced; none is code-complete here.

## 1. Landlock filesystem enforcement (closes class-4 path races + shrinks TCB)

**Target.** Move filesystem-path enforcement out of the supervisor's own path
code and into the kernel via Landlock. The supervisor grants a ruleset (allowed
directories, access rights); the kernel resolves and enforces race-free. This
simultaneously closes the symlink/magic-link/FUSE race surface (the kernel walks
the path) and removes our racy path logic from the trusted set.

**Why it is v1.10, not v1.9.2.** It is a model change: enforcement authority for
paths shifts from "supervisor inspects a copied path then performs the open" to
"kernel enforces a ruleset we installed." The two must agree, and the migration
needs a window where both run and are diffed.

**Soundness to preserve.** Landlock is allow-by-ruleset (default-deny within a
domain) — keep it that way. A path not granted by a rule is denied. Combine with
the v1.9.2 seccomp baseline (Landlock does not cover non-filesystem effects).
Kernel >= 5.13 for Landlock; >= 6.7 for the richer access rights — gate on a
runtime capability probe and fall back to supervisor-performs `openat2` with
`RESOLVE_*` flags where Landlock is unavailable.

**Skeleton.** `warden_landlock.{h,c}` (stub): `landlock_create_ruleset`,
`landlock_add_rule` per granted path, `landlock_restrict_self` after
`PR_SET_NO_NEW_PRIVS`. Build the ruleset from the same grant table the
fd-provenance invariant uses.

## 2. Acquisition-tiered mediation, formalized (judgment call 1)

**Target.** Make the three-tier split explicit and measured: (a) in-kernel BPF
fast path for scalar allow/deny, (b) unotify supervisor round-trip only for
pointer/path acquisition syscalls, (c) nothing on the use path. Today the split
exists implicitly (ALLOW vs NOTIFY in the baseline); v1.10 documents the latency
budget per tier and adds a benchmark gate so a regression that pushes a use-path
syscall into mediation is caught in CI.

**Soundness to preserve.** A capability cannot be acquired without passing tier
(a) or (b). The use path is unmediated *only* for fds already authorized through
acquisition — the fd-provenance invariant is the load-bearing premise; if it
weakens, tier (c) is no longer safe.

## 3. Post-grant re-mediation as an optional high-assurance mode (class 8)

**Target.** For customers who need it, a configurable mode that re-checks the
policy on *use* of a durable capability, not only on acquisition — defaulted
**off** so the common case keeps sub-microsecond use-path latency. Implementation
is use-path mediation (route `read`/`write`/`send`/`recv` on flagged fds back to
the supervisor) scoped to capabilities the policy marks high-assurance.

**Soundness to preserve.** Off by default must not mean "unsound by default":
the default already contains class 8 via narrow, provenance-tracked grants. The
mode adds assurance, it does not patch a hole.

## 4. UNKNOWN at the kernel edge — escalation ladder (judgment call 3)

**Target.** Define what a kernel-edge UNKNOWN does, with a policy-configurable
ladder defaulting conservative:

1. **deny** the individual syscall with `EPERM`, deterministically and within the
   resource bound, emit the v1.9.1 UNKNOWN diagnostic, let the agent degrade —
   **do not kill the process** (default).
2. **terminate the task**, not the process (high-assurance).
3. a **bounded** hold for an out-of-band re-decision — only if policy explicitly
   opts in and sets a timeout, after which it denies.

**Guards (all mandatory).** UNKNOWN must *never* silently become allow at the
edge. Repeated UNKNOWN on the same action trips the bounded-refusal breaker
(v1.8.2) so an agent cannot spin probing a boundary. Rung 3 must stay bounded or
it reintroduces the availability DoS and a human-in-the-loop dependency that
breaks the v1.9 HOOTL certification.

**Special case (recovers utility without weakening soundness).** When UNKNOWN
arises *because* the verifier could not safely inspect a pointer argument, route
to the supervisor-performs path (which can decide safely on a copied, validated
value) *before* falling through to deny.

## 5. Shrink the TCB to {minimal supervisor, small proof-checker, kernel} (judgment call 2)

**Target.** The trusted set should be the smallest thing that can enforce the
decision — and the decision procedure should *not* be in it.

- The supervisor carries **no policy logic**: it receives a notification, asks
  the verifier, enforces the verdict. That minimality is what makes it auditable.
- Move the SMT decision procedure **out** of the TCB via **proof-checking**:
  trust a tiny checker that validates the solver's proof object, not the solver
  itself. UNSAT/validity results carry a checkable certificate; the checker is
  the trusted artifact, the solver is not.
- Move filesystem enforcement **into** the kernel via Landlock (item 1), so we
  trust a well-audited kernel subsystem instead of our own path code.

**Honesty about the gap.** Per the current TCB document, the solver and compiler
are *today* in the trusted set. The line above is where we are driving it, not
where it sits. The audit ask we are working toward is "review a small supervisor
plus a small checker," not "trust a giant solver."

## Sequencing

1. Land item 4 (UNKNOWN ladder) — it is mostly verifier/Warden glue over
   existing v1.9.1 diagnostics and v1.8.2 breaker; lowest risk, high audit value.
2. Land item 1 (Landlock) behind a capability probe — biggest TCB and class-4
   win.
3. Item 2 (tiering + CI latency gate) — formalize and protect what exists.
4. Item 3 (re-mediation mode) — opt-in, contained, lowest urgency.
5. Item 5 (proof-checking) is research-grade and longest-horizon; pursue in
   parallel as the SMT proof-object pipeline matures.

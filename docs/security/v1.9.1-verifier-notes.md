# VAREK v1.9.1 — Verifier-Side Notes (UNKNOWN diagnostics, resource bounds)

These two changes touch the decision-procedure layer rather than the Warden.
They are specified here against the public verdict contract; integrate into the
verifier and confirm the existing verdict test suites are unaffected (both
changes are additive and must not alter any SATISFIED/UNSATISFIED outcome).

## 1. UNKNOWN-reason diagnostics

UNKNOWN remains the safe residue and its disposition is unchanged: it fails
closed and is never coerced to a pass. v1.9.1 makes it *navigable* by attaching
a reason, so an operator can see why an action was not cleared without waiting
for the v1.10/v1.11 fragments that will actually shrink the region.

The verdict carries an optional diagnostic only on UNKNOWN:

    typedef enum { VK_SATISFIED, VK_UNSATISFIED, VK_UNKNOWN } vk_verdict;

    typedef enum {
        VK_UNK_NONE = 0,
        VK_UNK_UNDISCHARGED_PREDICATE, // a predicate left the supported fragment
        VK_UNK_RESOURCE_BOUND,         // a deterministic bound was hit (see §2)
        VK_UNK_UNSUPPORTED_THEORY,     // outside any in-scope fragment
    } vk_unknown_reason;

    typedef struct {
        vk_unknown_reason reason;
        const char *predicate;   // the specific obligation left undischarged
        const char *fragment;    // the fragment that would resolve it, if any
                                 // (e.g. "bounded-string", "bitvector"), else NULL
    } vk_unknown_info;

Contract:
- Populated ONLY when verdict == VK_UNKNOWN. SATISFIED and UNSATISFIED carry no
  diagnostic and are byte-for-byte unchanged.
- `fragment` names the planned/forthcoming fragment that would move the case to a
  provable verdict (forward reference to v1.10/v1.11), or NULL if none applies.
- Soundness invariant unchanged: nothing here can turn an unsafe action into
  SATISFIED. The diagnostic is descriptive metadata on a refusal, not a decision.

Example (path predicate outside the current string support):

    verdict = UNKNOWN
    reason  = undischarged-predicate
    predicate = "path_has_prefix(arg.path, \"/srv/agent/\")"
    fragment  = "bounded-string"   // resolved by the v1.10 bounded string fragment

## 2. Deterministic resource bounds

The decision procedure is worst-case super-polynomial; an input that forces a
timeout must not hang the agent and must not be allowed to silently pass. v1.9.1
adds explicit, deterministic bounds. A bound hit yields UNKNOWN (fail closed)
with `reason = VK_UNK_RESOURCE_BOUND`.

Configuration (deployment-set, recorded in the audit log per query):

    max_solver_ms        // wall-clock ceiling per obligation
    max_obligation_steps // procedure-internal step ceiling (determinism anchor)
    memo_cache_entries   // size of the obligation memoization cache

Rules:
- Bounds are checked deterministically; the step ceiling, not wall-clock, is the
  authoritative cut so verdicts are reproducible across hardware. `max_solver_ms`
  is a secondary safety net for pathological cases.
- On any bound hit: return UNKNOWN. Never return SATISFIED on a bound hit.
- Memoization is keyed by a structural hash of the obligation; identical
  obligations resolve from cache. The cache changes latency only, never the
  verdict.

Operational guidance: publish the solver-time distribution (P99/P999) under an
adversarial corpus, not the median. The v1.10 verdict-distribution harness is the
right place to measure it; until then, record per-query solver time in the audit
log so the tail is observable.
